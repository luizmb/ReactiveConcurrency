# Part A — Concurrency, locks, hangs

## A0. Locking architecture (reference)

- `Locked<Value>` (`Sources/ReactiveConcurrency/Core/Locked.swift:7`) is the only mutex: `final class @unchecked Sendable` wrapping `NSLock` + value, `withLock((inout Value) -> R)`. **Non-reentrant.** Body is non-async → no suspension inside critical sections; the only risks are synchronous re-entry and synchronous blocking work under the lock.
- Lock owners:
  - `AnyCancellable._executed: Locked<Bool>` (`Core/Cancellable.swift:9`) — run-once guard; `_cancel()` executes **outside** the lock (`:21`). Safe.
  - `Diagnostics.config` (`Core/Diagnostics.swift:18`) — handler invoked outside the lock via snapshot (`:50-52`). Safe.
  - `SubjectCore._state` (`Subjects/SubjectCore.swift:15`), `CurrentValueCore._state` (`Subjects/CurrentValueSubject.swift:50`) — subscriber dict + latched completion. `send`/`complete`/`subscribe`/`unsubscribe` yield/finish continuations **while holding the lock** — safe because every continuation is `AsyncStream.makeStream()` default **`.unbounded`**: `yield`/`finish` never suspend and never run user code synchronously. The dict-stored continuations have **no** `onTermination`; only the outer per-subscriber streams (in `eraseToPublisher`/`share`) do, and those fire on the forwarding Task, never inside a core lock.
  - `SharedState._state` (`Operators/Publisher+Share.swift:35`), `AutoconnectState._connection` (`:81`).
- Everything else (operators, Deferred types, transformers) is lock-free: structured via `AsyncStream` + `Task`/`withTaskGroup`.
- The only cross-lock nesting in the library: `SharedState._state → SubjectCore._state`, in both `subscribe` (`Publisher+Share.swift:43-61`) and `unsubscribe` (`:65-72`). Order is consistent in both directions of use; `SubjectCore` never calls back into `SharedState`; publisher graphs are DAGs, so no inversion is constructible. **No cycle.**
- No `DispatchSemaphore`, `.wait()`, or condition variables anywhere in `Sources/`.

## A1. `Publisher.zip` is left-driven — hang + Combine divergence (HIGH, likely test-hang cause #2)

`Operators/Publisher+Combine.swift:79-98`:

```swift
for await sr in selfStream {
    ...
    guard let or = await otherIter.next() else { raw.finish(); return }
```

`zip` iterates `self` and only polls `other` after `self` emits. Consequences:

- `self` silent + `other` **fails** → failure never observed → downstream hangs. Combine's zip forwards a failure from either side immediately.
- `self` silent + `other` **finishes empty** → completion never observed → hangs. Combine completes as soon as either side definitively can't form a pair.
- Test shape that hangs: `neverEmittingSubject.eraseToPublisher().zip(.fail(e))` awaited to completion.

Same one-sided pull pattern exists in `DeferredStream` zip/apply/liftA2 (`Deferred/DeferredStream+Applicative.swift:36,62,91,116,138`) — acceptable there as plain sequence-zip semantics, but the `Publisher` form must match Combine.

**Fix**: restructure `Publisher.zip` on the combineLatest FIFO pattern (`Publisher+Combine.swift:150-216`): two `StreamBox`es, `withTaskGroup`, per-side FIFO buffers of unmatched values; emit a pair when both buffers non-empty; on `.failure` from either side forward immediately; complete when a side is finished **and** its buffer is empty. Regression tests: zip(silent, failing), zip(silent, empty), zip preserves pairwise order under interleaving.

## A2. Tests await natural completion with no time bound (HIGH, likely test-hang cause #1)

- Zero `.timeLimit` traits across all 337 tests (grep confirmed). One hang blocks the whole suite with no diagnostic.
- Pattern: `for await r in <op>._stream { ... }` / `pub.values` with no cancellation or timeout — `Tests/ReactiveConcurrencyTests/ClockAndTimingTests.swift:49,120,247,279`; `MulticastBufferMergeTests.swift:18-24` (collect helper).
- All timing operators delegate to **Hourglass** (`Operators/Publisher+Timing.swift`). If a Hourglass operator fails to finish its downstream, the awaiting test parks forever. Top suspect: `collect(every:clock:)` driven by `ImmediateClock` (`ClockAndTimingTests.swift:277-286`) — an immediate clock can livelock a window-flush loop. The culprit would live in Hourglass, not this repo — needs a focused reproduction.
- Secondary: `DeferredTaskTests.swift:208,248,291` rely on `race`/`alt` cancelling 5-second `Task.sleep`s (`Deferred/DeferredTask+Race.swift:35-41`, `DeferredTaskTOptional+Alternative.swift:20-24`). Correct today; if cancellation propagation ever regresses these become silent 5s stalls that read as hangs.

**Fix**: add `.timeLimit` traits to every suite; bound completion-awaits with Hourglass timeout or TestClock advancement; write a standalone reproduction for `collect(every:) + ImmediateClock` and chase into Hourglass if it reproduces.

## A3. Work performed under `Locked` in share/autoconnect (MEDIUM, hygiene / stall risk)

- `SharedState.subscribe` (`Operators/Publisher+Share.swift:43-61`): under `_state.withLock` it (1) runs `_upstream._stream.factory()` — arbitrary composed-publisher construction code, (2) spawns the upstream `Task`, (3) calls `_core.subscribe()` (nested SubjectCore lock).
- `SharedState.unsubscribe` (`:65-72`): under the lock it calls `state.upstreamTask?.cancel()`. `Task.cancel()` synchronously runs `AsyncStream` termination/cancellation handlers **on the calling thread**; in this library those handlers take locks (`share()`'s own `raw.onTermination` → `shared.unsubscribe` → SharedState+SubjectCore locks; subject publishers → `core.unsubscribe`). Today ordering is consistent (upstream direction only) so no deadlock is constructible, but this is one refactor (or one feedback-loop publisher) away from a real one.
- `AutoconnectState.connectOnce` (`:83-88`): runs `connect()` (factory + Task spawn, `:127-140`) under `_connection`'s lock.
- Practical impact today: a slow/blocking upstream factory stalls every concurrent `subscribe`/`unsubscribe` on that share — including teardown paths from `AnyCancellable.deinit` — on arbitrary threads.

**Fix pattern**: only bookkeeping inside `withLock`. Compute a decision + capture what to act on under the lock; run `factory()`, `Task` spawn, and `cancel()` after release. E.g. `unsubscribe`: `let toCancel: Task<Void, Never>? = _state.withLock { ... return task }; toCancel?.cancel()`.

## A4. `buffer` can drop a `.failure` (LOW)

`Operators/Publisher+Buffer.swift:17-37`: the stream buffers `Result<Output, Failure>` values, so under `.dropOldest` (`.bufferingNewest`) a buffered `.failure` element can be evicted by later successes → downstream sees `.finished` instead of the failure. Combine never drops completion events.

**Fix**: keep the bounded policy for successes but carry failure out-of-band (e.g. `Locked<Failure?>` set by producer, checked when the loop ends), or intercept before buffering.

## A5. `switchToLatest` truncates the active inner on outer completion (LOW, semantics)

`Operators/Publisher+FlatMap.swift:97-123`: when the outer loop ends normally it runs `innerTask?.cancel(); raw.finish()` — killing the still-running latest inner publisher. Combine completes only after the outer **and** the last inner complete. Values silently lost.

**Fix**: on outer completion, `await innerTask?.value` before `raw.finish()` (keep the cancel path for failures/termination).

## A6. Benchmarks `@unchecked Sendable` data races (LOW, benchmark-only, UB)

`Sources/Benchmarks/Benchmarks.swift:284-289`: `Sum` (`var v = 0; v &+= x` mutated from `@Sendable receiveValue` closures at `:95,107,141,156` with no synchronization), `CancellableBox`, `CombineBox` — unsound `@unchecked Sendable`. The adjacent `Counter` (`:275`) does it right with NSLock.

**Fix**: use `Locked` in all three.

## A7. Reviewed and confirmed safe (don't re-litigate)

- `flatMap(maxPublishers:)` token semaphore (`Operators/Publisher+FlatMap.swift:16-52`): `AsyncStream<Void>` pre-seeded with N tokens, `bufferingNewest(N)`; inner returns token in `defer`; `onCancel { slotsCont.finish() }` unblocks a parked `next()`. Balanced; no deadlock; a never-completing inner holding a slot is intended backpressure.
- `combineLatest` (`Publisher+Combine.swift:150-216`): StreamBox iterators advanced FIFO — a new `.fromA`/`.fromB` child is added only after the prior event is processed (`:185,196`), so at most one outstanding `next()` per box, matching StreamBox's `@unchecked Sendable` justification (`Core/StreamBox.swift:6`). Group-body `return` cancels/drains children (AsyncStream honors cancellation). Completes correctly if a source finishes before ever emitting (`:202-207`).
- Subject reentrancy: structurally impossible — `send` only buffers into unbounded continuations; subscriber code runs later on the forwarding Task (`Core/Diagnostics.swift:8-11` documents this; verified).
- `AnyCancellable`: deinit→cancel idempotent; `_cancel()` outside its lock; no re-entry into `_executed`.
- Sink cancellation hygiene (`Core/Publisher+Sink.swift:11-30`): pre-delivery `guard !Task.isCancelled` prevents delivery of already-buffered values after cancel; completion callback suppressed after cancel (`:26-28`).

# Part B — Typeclass lawfulness & algebra

## B0. Instance catalogue (reference)

Base types:

| Type | Functor | Applicative | Monad | Alternative |
|---|---|---|---|---|
| `DeferredTask` | ✓ | `pure` + sequential `apply` — **lawful, ap≡bind** | sequential ✓ | — (race lives in T-stacks) |
| `DeferredStream` | ✓ | `pure` single-shot + **zip** apply | **concatMap** (lawful list monad) | `alt`=concat (no `empty` defined) |
| `Publisher` | ✓ | `pure`=`just` + **zip** apply | **merge** (unordered); `flatMap(maxPublishers:1)` sequential | `empty()` + `alt`=concat ✓ lawful monoid |

Transformer stacks (× DeferredTask/DeferredStream/Publisher): TEither, TResult, TOptional, TArray, TValidation, ReaderT, WriterT, StatefulT. Symbolic operators in `ReactiveConcurrencyOperators` are all thin delegates to the named functions — **no drift**. Functor laws hold everywhere. Purity: clean — no `Date()`/`UUID()`/ambient state; every `Task {}` in transformers sits inside a `DeferredStream` factory, so laziness is preserved (`DeferredStream.swift:59-61`, `DeferredTask.swift:79`, `Publisher.swift:27-56`).

## B1. Applicative identity broken for streaming types (HIGH)

- `applyDeferredStream` (`Deferred/DeferredStream+Applicative.swift:105-124`) — zip-pairwise; `pure` yields one element then finishes (`:5-11`).
- `applyPublisher` (`Operators/Publisher+FP.swift:64-71`) = `fns.zip(values).map { $0.0($0.1) }`; `pure = just` (`:41-43`).

Identity law `pure(id) <*> v == v` fails for |v| > 1: zip truncates at the shorter side → `[1,2,3]` becomes `[1]`. Classic ZipList defect — a zippy applicative needs `pure` = infinite repeat. Homomorphism and interchange break the same way. `seqLeft`/`seqRight` (`Publisher+FP.swift:46-53`, `DeferredStream+Applicative.swift:14-21`) are zip-based and inherit it.

## B2. Applicative ≠ Monad for streaming types (HIGH, design decision required)

Consistency law `apply(f, a) == f.flatMap { g in a.map(g) }`:

- `DeferredStream.flatMap` = concatMap (`Deferred/DeferredStream+Monad.swift:6-22`, comment says "concatMap: sequential, lawful" — true as a monad). Derived ap = cartesian. Counterexample: fns=[f,g], values=[x,y] → zip-ap `[f(x), g(y)]` vs monad-ap `[f(x), f(y), g(x), g(y)]`.
- `Publisher.flatMap` = merge (`Operators/Publisher+FlatMap.swift:60-93`): cardinality differs from zip-ap AND ordering is non-deterministic — as an ordered-stream monad, associativity only holds up to unordered-bag equality. `flatMap(maxPublishers: 1)` (`:8-54`) recovers sequential lawful ordering (and is what ReaderT/WriterT-over-Publisher use).
- `DeferredTask` is the only consistent one: `applyDeferredTask` (`Deferred/DeferredTask+Applicative.swift:34-43`) runs fns then values, exactly `flatMap`+`map` order. ✓

**Recommended resolution** (option b): keep zip as the stream product — it's the semantics users actually want — but stop calling it Applicative. Rename/document `apply`/`liftA2`/`seqLeft`/`seqRight` for streams as zippy Semigroupal (`zip`/`zipWith`); either derive `<*>` from `flatMap` or stop exposing `<*>`+`pure`-as-Applicative for `DeferredStream`/`Publisher`. Option (a) — cartesian apply — is lawful but rarely wanted for streams. Whichever is chosen, `ReactiveConcurrencyOperators` `<*>`/`*>`/`<*` delegation must follow.

## B3. WriterT bind drops the continuation's log — REAL BUG (HIGH)

- `WriterTDeferredTask+Monad.swift:17-19`, `WriterTDeferredStream+Monad.swift:17-21`, `WriterTPublisher+Monad.swift:16-18`: `flatMapT` returns `Writer(<inner effect>, outerLog)` — `fn(a).log` is **discarded**. Doc comments claim logs are appended/concatenated — the code contradicts them.
- Left identity fails: with `f = { Writer(pure(b), ["log"]) }`, `pure(a) >>= f` yields log `[]`/outer while `f(a)` yields `["log"]`.
- **Tests enshrine the bug**: `Tests/ReactiveConcurrencyTests/WriterTDeferredTask+Tests.swift:19-27` (`flatMapTPreservesOuterLog`, inner `["inner"]`, asserts `== ["outer"]`) and `:29-38` (`flatMapTChains`, inner `["b"]`, asserts `== ["a"]`).
- Structural cause: the shape `Writer(Deferred*<A>, W)` keeps the log **outside** the async effect, so bind cannot see the continuation's log without running the effect.

**Fix**: re-shape WriterT to the standard effect-carrying-pair: `DeferredTask<(A, W)>`, `DeferredStream<(A, W)>`, `Publisher<(A, W), F>`. Bind then combines logs inside the effect: run outer → `(a, w1)`, run `fn(a)` → `(b, w2)`, return `(b, w1 <> w2)`. Breaking change; rewrite the two tests above to assert `outer <> inner`; update the WriterT operator files in `ReactiveConcurrencyOperators` (delegation only, signatures change).

## B4. Three incompatible `alt` semantics under one `<|>` (MEDIUM)

1. Base `DeferredStream.alt` (`Deferred/DeferredStream+Monad.swift:32-51`) and `Publisher.alt` (`Operators/Publisher+FP.swift:110-139`) = **concat**. With `Publisher.empty()` (`Core/Publisher.swift:141-143`) this is a lawful monoid ✓. `DeferredStream` has **no `empty`** — the monoid identity element is missing (alt itself associative). Add `DeferredStream.empty`.
2. `DeferredTaskTOptional.alt` / `TResult.alt` (`DeferredTaskTOptional+Alternative.swift:11-26`, `DeferredTaskTResult+Alternative.swift:11-29`) = **concurrent race**, first non-nil/`.success` wins. Violates Alternative left-bias (`alt(pure(1), pure(2))` may be 1 **or** 2 depending on scheduling) and referential transparency; double-failure returns a scheduling-dependent "last" failure.
3. `PublisherTOptional.alt` / `TResult.alt` (`PublisherTOptional+Alternative.swift:13-18`, `PublisherTResult+Alternative.swift:11-21`) = **zip-truncate**, left-biased per pair. Deterministic but not monoidal: right identity fails for |x|>1 (zip truncates), and no `emptyT` is even defined.

**Fix**: keep the race combinators but rename them `race` (they're valuable, just not Alternative); give Publisher-T stacks a lawful element-wise-over-concat alt or drop them; reserve `<|>` for instances where the monoid laws hold.

## B5. Zero algebraic law tests (HIGH leverage)

- grep for `identity|associativ|homomorph|interchange|law` across `Tests/` → 0 hits. All 337 tests are example-based; `FPParityTests.swift` checks shapes/values, not laws.
- The only law-adjacent tests (WriterT) assert the violation (B3).
- Consistency law (`ap == flatMap+map`) tested nowhere; multi-element Applicative identity tested nowhere — either would have caught B1/B2 immediately.
- Coverage asymmetries: StatefulT/WriterT have tests for Task & Stream but not Publisher; Alternative tests check happy paths only, never left-bias/timing.
- StatefulT has **no Monad** for any stack (documented `inout`-across-await limitation; consistent — operators ship only Functor+Applicative) and `StatefulTPublisher+Monad` doesn't exist at all. Document as intentional.

**Fix**: add a law-test harness: generic helpers that run a structure to a collected `[Element]`/value and assert equality, then instantiate per instance: Functor id/composition; Applicative id/homomorphism/interchange; Monad left/right identity + associativity; ap-consistency; Alternative monoid (left/right identity, associativity); Validation accumulation (double-failure combines, order preserved). Run each over DeferredTask, DeferredStream, Publisher, and every transformer stack.

## B6. Minor

- `applyT*Validation` re-inlines the 4-case Validation switch instead of delegating to `Validation.liftA2` (`DeferredTaskTValidation+Applicative.swift:23-37`, `PublisherTValidation+Applicative.swift:18-30`, stream variant) — duplicated semantics kept in sync by hand; delegate instead. Accumulation itself is CORRECT (`E.combine(e1, e2)` on double failure) ✓.
- `PublisherTValidation` applicative composes over `Publisher.zip` → inherits A1's hang behavior.

# Part C — Combine parity & subscription lifetime

## C0. Architectural deltas (intentional, document-level)

- `Publisher<Output: Sendable, Failure: Error>` is a concrete **struct** backed by `DeferredStream<Result<Output, Failure>>` (`Core/Publisher.swift:12`), not a protocol. No `Publishers.*` namespace, no `Subscriber`/`Subscription` protocols, no custom conformances — extension point is `Publisher.init(body:)` (`:27-56`) + bridges.
- `AnyPublisher` = typealias; `eraseToAnyPublisher()` is a no-op (`:125-130`).
- Cold-by-default everywhere; `DeferredStream` subsumes Combine's `Deferred`.
- **No demand/backpressure model** (`Subscribers` enum only holds `Completion`, `Core/Subscribers.swift`). All interior operator AsyncStreams use default `.unbounded` buffering — only `buffer()` and `flatMap(maxPublishers:)` set a policy (grep-verified).

## C1. Parity table (condensed; ✅ present, 🔁 renamed/reshaped, 🟰 different semantics, ❌ missing, ➕ added)

- Creation: Just/Empty/Fail 🔁 statics (`Publisher.swift:135,141,156`); Future 🔁🟰 **cold**, async-closure (`Publishers/Future.swift`); Record ✅; Sequence ✅; Result/Optional `.publisher` ✅ (`Publishers/Convenience.swift`); Timer 🟰 Clock-based cold (`Publishers/Timer.swift:9`); Deferred ❌ (subsumed); `@Published` ❌.
- Transform: map(+keypaths)/mapError/compactMap/replaceNil/setFailureType/scan/reduce/collect(+time) ✅; tryMap/tryScan/etc 🟰➕ **typed throws + Result-returning forms** (`Operators/Publisher+Try.swift` throughout); flatMap(+maxPublishers)/switchToLatest ✅; encode/decode 🔁 closure-based, not TopLevelEncoder/Decoder (`Publisher+Map.swift:206-235`).
- Filtering: filter/removeDuplicates/first/last/prefix/drop/output/ignoreOutput/replaceEmpty + try-variants ✅ (`Publisher+Filtering.swift`, `+Try.swift`); `prefix(untilOutputFrom:)` ❌.
- Combining: merge (arity ≤4 + MergeMany) ✅; combineLatest/zip (2–4 + transforms) ✅; prepend/append ✅.
- Timing: delay/debounce/throttle/timeout/measureInterval 🟰 all require explicit `Clock` (Hourglass); `timeout` requires an error (no default-completion variant).
- Errors: catch/tryCatch/replaceError/retry(n)/assertNoFailure ✅; infinite `retry()` ❌.
- Scheduling: receive(on:)/subscribe(on:) 🟰 take `any Actor` (`Operators/Publisher+Scheduling.swift:14,37`); no Scheduler/RunLoop/DispatchQueue.
- Sharing: share/multicast/makeConnectable/connect/autoconnect ✅ (`Operators/Publisher+Share.swift`); buffer 🟰 custom `BufferStrategy`, no prefetch/customError.
- Consuming: sink (both) ✅; assign(to:on:) 🟰 **weak** object capture (`Operators/Publisher+Assign.swift:14-56`); assignOnMain ➕; handleEvents ✅ (no receiveRequest); print/breakpoint 🟰 assertionFailure not SIGTRAP (`Publisher+Debug.swift`).
- Subjects: PassthroughSubject/CurrentValueSubject (+sync `.value`) ✅; `Subject` protocol reshaped (send/send(completion:)/eraseToPublisher); AnySubject ➕.
- Modern touch (added): `.values`/`.results` AsyncSequence bridges (`Core/Publisher+Bridges.swift:24,33`), `firstValue()`/`firstResult()` + lazy `DeferredTask` forms (`Publishers/Future.swift:65-104`), DeferredTask↔Publisher bridges (`:42-59`), FP algebra + operators + transformer targets, `Continuation` with `yieldAll`/`suspendUntilCancelled` (`Core/Publisher.swift:68-120`), opt-in `Diagnostics`, full Linux/Windows/Android support (no Combine/Apple-only APIs).

## C2. Subscription lifetime (verified Combine-faithful ✓)

- Drop-the-cancellable cancels: `AnyCancellable.deinit { cancel() }`, idempotent via `Locked(false)`, `_cancel()` runs outside the lock (`Core/Cancellable.swift:9-24`).
- Retention chain: `AnyCancellable` → `_cancel` closure → `Task` → stream iteration → captured core. Sink keeps upstream alive while retained (`Core/Publisher+Sink.swift:10-30`).
- No facade retain cycles: subject `eraseToPublisher()` captures the **core**, not the subject object (`Subjects/PassthroughSubject.swift:21`, `CurrentValueSubject.swift:21`).
- `store(in:)` both overloads ✅ (`Cancellable.swift:31-37`).
- Sends after `sink()` returns are always delivered: `factory()` runs synchronously in `sink`, registering the continuation under lock before return (`Publisher+Sink.swift:10`, `SubjectCore.swift:17-33`). Per-subscriber FIFO preserved.
- Send/complete after completion: dropped + `Diagnostics.warn` (`SubjectCore.swift:50-51,68-69`). Late subscribers to a completed subject get the stored completion immediately (`:20-27`). Matches Combine.

## C3. Semantic divergences to document (not bugs)

1. **`cancel()` is cooperative, not synchronous.** `task.cancel()` + pre-delivery `Task.isCancelled` guard (`Publisher+Sink.swift:13-17`) stops future deliveries promptly, but an in-flight delivery isn't interrupted — unlike Combine's synchronous teardown. Document explicitly.
2. **Subject delivery is asynchronous.** `send` buffers into continuations; sinks observe on their own Task. `subject.send(1); print("after")` prints before the sink sees 1. Acknowledged in README:224-228; upside: reentrancy anomalies structurally impossible.
3. **`assign(to:on:)` captures the object weakly** (Combine: strongly). Arguably better (no accidental retention) — but undocumented; note it in README/DocC.
4. **`Future` is cold** and re-runs per subscription (Combine: eager, shared result). Intentional (`Publishers/Future.swift:3-8`).

## C4. README/doc overclaims to fix

- README:15-16 "Every `Publisher` is an `AsyncSequence` under the hood" — `Publisher` does **not** conform to `AsyncSequence` (`Publisher.swift:12`); iteration goes through `.values`/`.results`. Fix prose (examples already use `.values` correctly).
- README:139-143 "a slow `for await` loop naturally slows production — backpressure for free" — overstated: interior streams are `.unbounded`; a slow consumer causes unbounded buffering at the final stage, and cold sources (`sequence` via `yieldAll`) flood their buffer on subscription. Real bounding needs explicit `buffer(size:)`. Rewrite the backpressure section honestly (or, larger effort, introduce bounded interior policies).
- README:569 `share()` "multicast via a `PassthroughSubject`" — implementation is a raw `SubjectCore` + refCount (`Publisher+Share.swift:26-73`). Wording.
- README:165-168 cancellation prose should state the non-synchronous cancel explicitly (C3.1).
